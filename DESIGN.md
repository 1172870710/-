# 像素沙盒 - 游戏设计框架

---

## 一、游戏定位

像素风 AI NPC 沙盒多人游戏。外观类似星露谷的田园小镇，但社交层面追求大胆深度：NPC 能自发产生爱恨情仇、背叛、谋杀、栽赃、三角恋等黑暗人性剧情，玩家可调查和介入。目标是让系统能涌现出**连开发者都意想不到的剧情**。

---

## 二、核心支柱

### 支柱 1：社交戏剧引擎（Social Drama Engine）
这是整个游戏的**心脏**，周期性扫描世界状态，识别 NPC 关系中的"戏剧潜力"，通过 LLM 生成事件并传播涟漪效应。

### 支柱 2：NPC 主观世界模型
一切内容物（道具、场景、事件）通过 `narrativeTags` 与 NPC 的内心状态产生关联。同一把铁锹对凶手/目击者/普通人的主观意义完全不同。

### 支柱 3：玩家调查系统
戏剧事件发生后，不同 NPC 知情的程度不同（全知/部分知情/碎片/一无所知），玩家需要通过对话、观察、推理来拼凑真相。

---

## 三、NPC 社交八维关系

```
A ←→ B 的关系由 8 个维度组成（每维度 0~1）：

  信任 — 高：托付秘密、借钱、作证
         低：处处提防、拒绝透露信息

  爱慕 — 高：追求、守护、牺牲
         低：冷漠、无视

  恐惧 — 高：回避、顺从、先下手为强
         低：不在乎、挑衅

  尊敬 — 高：追随、效仿、维护对方名誉
         低：鄙视、轻视对方意见

  嫉妒 — 高：中伤、破坏对方的好事、偷窃
         低：为对方成功高兴

  怨恨 — 高：复仇、暴力、谋杀
         低：能化解矛盾

  亏欠 — 高：愧疚、想要报恩
         低：无所谓、觉得理所应当

  怀疑 — 高：调查、试探、准备揭发
         低：轻信、容易上当
```

关系状态由 8 维组合决定：`陌生人 → 熟人 → 朋友 → 密友 → 爱慕 → 恋人`
反面：`陌生人 → 看不顺眼 → 对手 → 仇敌 → 死敌`

两种状态可以并存（如：又爱又恨）。

## 四、NPC 内心系统

```
NPC 内部结构：

┌─────────────────────────────┐
│ 人格（固定）                  │  ← 大五人格，生成时确定不再变
│ · 开放性 · 尽责性 · 外向性    │
│ · 宜人性 · 神经质            │
├─────────────────────────────┤
│ 信念（几乎不变）              │  ← 行为准则，极端事件下可被打破
│ · "家人至上" / "弱肉强食"     │
│ · "以德报怨" / "有仇必报"     │
├─────────────────────────────┤
│ 欲望（中期变）                │  ← 当前最强烈的驱动力
│ · 贪财 / 爱欲 / 权力 / 复仇    │  ← 随时间+事件动态切换
│ · 生存 / 自由 / 守护 / 名誉    │
├─────────────────────────────┤
│ 情绪（短期变）                │  ← 快速波动
│ · 快乐 · 愤怒 · 恐惧 · 惊讶    │
├─────────────────────────────┤
│ 压力值 (0~1)                 │  ← 累积型，高→心理崩坏
│ · 被威胁、被孤立、长期贫穷...  │
├─────────────────────────────┤
│ 心理防线（底线）              │  ← 被突破→极端行为
│ · "家人受威胁"→暴怒            │
│ · "被公开羞辱"→极端报复        │
│ · "秘密被揭发"→杀人灭口        │
└─────────────────────────────┘

心理状态转换：
  正常 ──压力↑→ 焦虑 ──压力↑→ 偏执 ──压力↑→ 崩溃 ──压力↑→ 狂暴
```

## 五、戏剧引擎

### 5.1 工作流程

```
每 5-10 分钟真实时间运行一次：

  ① 扫描全 NPC 关系图
     识别"戏剧潜力"：哪些关系组合具备了冲突条件
     · A 恨 B 且 A 压力高 → 冲突潜力
     · A 爱 B 但 B 爱 C → 三角潜力
     · A 有秘密 且 B 在调查 → 灭口潜力
     · A 贪财 且 B 有钱 且 A 疑 B → 犯罪潜力

  ② 匹配戏剧模式（DramaPatterns）
     从预定义模式库中筛选触发条件达标者

  ③ LLM 生成事件种子
     具体化事件：谁、做了什么、动机、谁目击、什么证据

  ④ 事件链传播
     涟漪式影响所有相关 NPC（恐惧、复仇欲、道德困境...）

  ⑤ 反馈到 NPC 行为和对话
     NPC 的行为/对话/商店都因事件而变化
```

### 5.2 戏剧模式库（可扩展）

```
爱情类：单向暗恋、秘密恋情、三角恋、病态控制
利益类：背叛、敲诈、经济欺骗、合谋陷害
暴力类：预谋杀人、激情杀人、连环作案、复仇处刑
心理类：精神崩溃、偏执发狂、人性腐化、邪教疯魔
悬疑类：失踪迷案、栽赃陷害、冒牌顶替、失忆揭露
连锁类：复仇链、群众私刑、小镇崩坏
```

## 六、事件可见性系统

```
事件发生后，不同 NPC 的知情程度分层：

  全知（凶手自己）
    → 知道全部真相，主动撒谎、销毁证据、伪装

  部分知情（目击者）
    → 知道部分信息，但可能因恐惧/利益选择沉默

  碎片信息（间接相关者）
    → 知道某个片段（听到争吵声/看到买了铁锹/注意到不在场证明矛盾）

  一无所知（其他 NPC）
    → 只知道"有人不见了"/"镇上气氛不对"

玩家调查过程：
  发现异常 → 询问 NPC → 对比证词 → 搜集物证 → 推理 → 揭发/包庇/私了
```

## 七、道具/场景的意义系统

```
所有内容物注册时带有 narrativeTags，戏剧引擎自动匹配：

物品 = 客观属性 + 叙事标签

  铁锹：['tool', 'weapon_blunt', 'digging', 'concealable', 'heavy']
  信件：['evidence', 'secret', 'personal', 'readable']
  毒药：['poison', 'lethal', 'stealthy', 'alchemy', 'illegal']
  婚戒：['love', 'commitment', 'betrayal', 'expensive', 'personal']

场景 = 客观属性 + 叙事标签

  废弃矿洞：['dark', 'isolated', 'hidden', 'dangerous', 'secrets']
  镇广场：['public', 'safe', 'witnessed', 'social']
  后山树林：['secluded', 'dark', 'natural', 'hidden']

事件发生后，"污染"物品和场景的意义：
  · 用来杀人的铁锹 → 凶器（对知情人）
  · 埋过尸体的树林 → 恐惧场所（对目击者）
  · 关键时间点 → 被 NPC 在对话中回避或撒谎
```

## 九、内容注册统一接口

以后添加新内容只需注册三类信息，系统自动融入戏剧网络：

```javascript
// 道具
ItemRegistry.register({ id, name, type, narrativeTags, ... });

// 场景
LocationRegistry.register({ id, name, narrativeTags, ... });

// NPC
NPCFactory.create({ name, job, personality, narrativeTags, ... });
```

`narrativeTags` 是连接世界的"语义词"，戏剧引擎根据标签匹配事件参与者、场景和关键道具。

## 十、技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 客户端 | Godot | 跨平台、性能好、原生 WebSocket |
| 网络协议 | 纯 WebSocket (ws) | Godot 原生支持，无需适配 |
| 消息格式 | JSON（开发期）→ 二进制 | 先快后优 |
| 存档格式 | JSON 文件 → SQLite | 先简单后可靠 |
| LLM | DeepSeek API | 已有 Key，性价比高 |
| 模块通信 | EventBus | 解耦，方便测试和扩展 |
| 服务端模块 | ESM (import/export) | 现代化，新文件统一 |

## 十一、LLM 在系统中的角色

LLM 不是 NPC 行为的"执行者"，而是"编剧顾问"：

1. **戏剧编排**：扫描关系图，识别有趣组合，生成事件梗概
2. **对话深度**：NPC 对话基于上下文、秘密、谎言（有破绽/说漏嘴）
3. **行为合理化**：确保 NPC 行为是基于人格+经历的一贯模式
4. **调查互动**：玩家提问时，NPC 根据知情程度和性格选择回答策略
